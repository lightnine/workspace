目标：使用现代前端框架从零实现一个类似 Databricks Workspace 的 UI 应用。

这个前端应用需要支持以下核心功能，并与后端（我等会给你后端API的协议规范）交互：

---

## 🧩 核心功能需求

1. **导航栏（Sidebar）**
   - 类似工作区左侧导航，可切换不同模块：
     - Workspace（文件与笔记本浏览器）
     - Recents（最近访问）
     - Search（搜索）
     - Compute
     - Jobs / Pipelines
     - SQL / Dashboards / Experiments
   - 点击导航更新主体视图内容展示

2. **工作区浏览器（Explorer）**
   - 以树状结构展示文件夹和文件（Notebook、脚本、文件、Dashboard 等
   - 支持右键或上下文菜单：
     - 新建文件／文件夹
     - 重命名
     - 删除
     - 移动
   - 支持拖拽排序与层级调整（可选）

3. **顶部搜索栏**
   - 输入关键词实时搜索 Workspace 中的对象（包括 Notebook、文件、Dashboard 等）

4. **文件／Notebook 编辑器**
   - 内置代码编辑器（基于 Monaco / CodeMirror）
   - 支持语法高亮、自动补全、文件保存、运行单元（Notebook），与后端 API 通信
   - 支持文件预览、内容 diff（可选）

5. **Tab 视图（多打开文件）**
   - 编辑器可同时打开多个标签页
   - 标签可拖动关闭，状态保存（如文档未保存提示）

6. **首页 Dashboard**
   - 显示 “Recents”/常用任务快捷入口
   - 近期访问的文件、Notebook、Dashboard 和搜索历史

7. **权限与用户菜单**
   - 顶部显示当前用户
   - 提供个人设置／登出入口
   - 展示对象权限 UI（查看／修改用户角色）

8. **通知与状态提示**
   - 提示操作成功、失败、加载状态等

---

## 🖥 交互细节与 UX

- 所有 API 调用应异步，并处理 loading、错误提示
- 树状浏览器能实时展开/收缩
- 编辑器与预览模式切换
- 搜索建议作为下拉展示
- 文件拖拽支持到目标目录并提示确认

---
